AnalyticsCategoryClientBehavior

public protocol AnalyticsCategoryClientBehavior

Behavior of the Analytics category that clients will use

  • disable()

    Stop sending analytics events. The exact behavior of this API is dependent upon the plugin, but at a minimum, the plugin must continue to accept analytics record events, to be sent when the client issues enable.

    Declaration

    Swift

    func disable()
  • enable()

    Begin (or resume) sending analytics events.

    Declaration

    Swift

    func enable()
  • record(_:)

    A convenience method to allow clients to record an AnalyticsEvent by specifying only the name. Internally, this method creates a BasicAnalyticsEvent with the specified name and no attributes or metrics.

    Declaration

    Swift

    func record(_ name: String)

    Parameters

    name

    The name of the event
  • record(_:)

    Records an AnalyticsEvent. This method is synchronous, but this protocol does not provide a guarantee of delivery. Instead, a successful return from this method means that the plugin has accepted the event and will deliver it according to the rules of that plugin.

    Declaration

    Swift

    func record(_ event: AnalyticsEvent)

    Parameters

    event

    The AnalyticsEvent to record
  • update(analyticsProfile:)

    Updates the metrics backend with the analytics profile associated with the current user on the current app installation. The exact meaning of an AnalyticsProfile varies by plugin. For example, in AWSPinpoint, an AnalyticsProfile maps exactly to an Endpoint–a union of user and device that uniquely identifies the user’s installation of your app on their specific device.

    Declaration

    Swift

    func update(analyticsProfile: AnalyticsProfile)
